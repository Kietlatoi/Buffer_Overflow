Giới Thiệu Buffer Overflow

🧠 Buffer Overflow là gì?

Buffer Overflow (Tràn bộ đệm) là lỗi bảo mật xảy ra khi một chương trình ghi dữ liệu vượt quá kích thước của vùng nhớ (buffer) đã được cấp phát.

Dữ liệu dư thừa sẽ ghi đè lên các vùng bộ nhớ lân cận, có thể làm hỏng dữ liệu quan trọng hoặc thay đổi luồng thực thi chương trình.

⚠️ Tại sao nguy hiểm?

  • 😈 Kẻ tấn công có thể thay đổi Return Address
  • 💻 Thực thi mã tuỳ ý (Arbitrary Code Execution)
  • 🔑 Leo thang đặc quyền (Privilege Escalation)
  • 💥 Gây ra Segmentation Fault hoặc crash
  • 📊 Nằm trong danh sách OWASP Top 10 lỗ hổng nguy hiểm nhất

Các Ví Dụ Code

Ví dụ 1 — Truy cập ngoài biên mảng
// vidu1.c
#include <stdio.h>
#include <string.h>

int main() {
    int array[5] = {1, 2, 3, 4, 5};
    // ❌ Chỉ số 5 hợp lệ là 0..4
    printf("%d\n", array[5]);
    return 0;
}

🔍 Giải Thích

Mảng array có 5 phần tử hợp lệ ở chỉ số 0 đến 4.

Việc đọc array[5] là truy cập ngoài biên (out-of-bounds) — đọc vùng nhớ không thuộc mảng.

⚠️ Kết quả không xác định (Undefined Behavior) — có thể in ra giá trị rác, crash, hoặc bị khai thác.
Ví dụ 2 — Ghi đè hàng loạt lên stack
// vidu2.c
#include <stdio.h>
#include <string.h>

int main() {
    int array[5];
    int i;
    // ❌ i chạy từ 0 đến 255!
    for (i = 0; i <= 255; i++) {
        array[i] = 10; // Ghi đè vượt quá bounds
    }
    return 0;
}

💥 Điều Gì Xảy Ra?

Vòng lặp chạy 256 lần trong khi mảng chỉ có 5 phần tử.

Từ array[5] trở đi, chương trình ghi đè lên:

  • 📌 Biến i trên stack → vòng lặp vô tận có thể xảy ra
  • 📌 Saved EBP
  • 📌 Return Address → Crash (Segfault)
🚨 Chương trình gần như chắc chắn crash — đây là Buffer Overflow cổ điển trên stack.
Ví dụ 3 — Khảo sát địa chỉ bộ nhớ & Stack Frame 
// vidu3.c
#include <stdio.h>
#include <string.h>

void function(int a, int b) {
    int array[5];
    // Hàm này để khảo sát stack frame
}

int main() {
    function(1, 2);
    printf("This is where the return address points");
    return 0;
}

🔎 Mục Đích

Ví dụ này không có bug thực sự — mục đích là dùng gdb để:

  • 🔬 Khảo sát địa chỉ bộ nhớ của các biến
  • 📊 Hiểu cấu trúc stack frame của hàm
  • 🗺️ Biết Return Address trỏ về đâu (printf trong main)
💡 Biên dịch với: gcc -fno-pie -no-pie -ggdb vidu3.c -o vidu3
⭐ Ví dụ 4 — Lỗ Hổng gets() [TRỌNG TÂM] 
// vidu4.c
#include <stdio.h>
#include <string.h>

void return_input(void) {
    char array[30];   // Buffer chỉ 30 bytes
    gets(array);      // ❌ KHÔNG kiểm soát độ dài!
    printf("%s\n", array);
}

int main() {
    return_input();   // Gọi hàm có lỗ hổng
    return 0;
}

🎯 Lỗ Hổng Chính

🔴
Hàm gets() cực kỳ nguy hiểm

Đọc input đến khi gặp \n — không giới hạn độ dài.

💀
Attack Vector

Nhập chuỗi dài hơn 30 ký tự để ghi đè Saved EBP và Return Address.

⛔ gets() đã bị loại khỏi chuẩn C11 vì lý do bảo mật. Dùng fgets(array, 30, stdin) thay thế.

1. Phân Tích Source Code

// vidu4.c — Annotated
#include <stdio.h>
#include <string.h>

void return_input(void) {
    // ① Buffer chỉ cấp phát 30 bytes trên stack
    char array[30];

    // ② gets() đọc KHÔNG giới hạn — LỖ HỔNG!
    gets(array);

    printf("%s\n", array);
}

int main() {
    // ③ Gọi hàm chứa lỗ hổng
    return_input();
    return 0;
}

① Buffer Allocation

char array[30] — 30 bytes được cấp trên stack của hàm return_input.

② Lỗ Hổng: gets()

Hàm gets(array) đọc từ stdin cho đến ký tự \n mà không kiểm tra độ dài.

Nếu nhập > 30 ký tự → tràn qua Saved EBP và Return Address.

③ Attack Vector

Nhập chuỗi đủ dài để ghi đè Return Address → điều hướng CPU tới địa chỉ bất kỳ.

💡 Mục tiêu của kẻ tấn công: biết địa chỉ buffer trên stack, ghi shellcode vào đó, ghi đè Return Address trỏ vào shellcode.

💥 Kịch Bản Tấn Công — Ghi Đè Return Address

1

📍 Xác Định Vị Trí Buffer trong Stack

lea -0x26(%ebp), %eax ; array bắt đầu tại EBP - 0x26

Từ lệnh assembly, ta suy ra:

  • 0x26 (hex) = 38 (decimal)
  • Buffer array nằm cách Saved EBP đúng 38 bytes
2

🗺️ Xác Định Các Địa Chỉ Quan Trọng (trước khi nhập)

OffsetĐịa ChỉGiá TrịÝ Nghĩa
+0x00xbffff5900x00000001Padding / biến cục bộ
+0x40xbffff5940xbffff654Padding / biến cục bộ
+0x80xbffff5980xbffff5a8⚠️ Saved EBP (EBP của main)
+0xC0xbffff59c0x08048478🎯 Return Address — mục tiêu tấn công!
3

🧮 Tính Toán Địa Chỉ Buffer Start

Saved EBP tại: 0xbffff598
Buffer offset: EBP - 0x26 = EBP - 38
Buffer Start = 0xbffff598 - 0x26 = 0xbffff572

🎮 Mô Phỏng Tương Tác — Nhập Payload và Xem Stack Bị Ghi Đè

Nhập chuỗi payload bên dưới để xem trực quan cách bộ nhớ stack bị ghi đè:

📦 Phân Tích Payload: A×20 + B×20 + C×10

AAAAAAAAAAAAAAAAAAAA
20 × 'A' (0x41) — Lấp đầy phần đầu Buffer (bytes 0–19)
BBBBBBBBBBBBBBBBBBBB
20 × 'B' (0x42) — Lấp tiếp Buffer + tràn vào Padding (bytes 20–39)
CCCCCCCCCC
10 × 'C' (0x43) — Ghi đè Saved EBP & Return Address (bytes 40–49)
Vùng Nhớ Địa Chỉ Bắt Đầu Độ Rộng Dữ Liệu Bị Ghi Trạng Thái
Buffer (khai báo) 0xbffff572 30 bytes 20 'A' + 10 'B' ✅ Hợp lệ
Padding (compiler) 0xbffff590 8 bytes 8 'B' ⚠️ Tràn nhẹ
Saved EBP 0xbffff598 4 bytes 2 'B' + 2 'C' 🚨 EBP bị hỏng
Return Address 0xbffff59c 4 bytes 4 'C' → 0x43434343 💀 MẤT KIỂM SOÁT

🔚 Điều Gì Xảy Ra Khi Hàm return_input Thực Hiện ret?

1

CPU thực hiện POP EIP — đọc Return Address từ stack (0xbffff59c)

↓
2

Return Address hiện tại là 0x43434343 (đã bị ghi đè bởi 'CCCC')

↓
3

CPU nhảy đến địa chỉ 0x43434343 — không phải vùng nhớ hợp lệ chứa code

↓
💀

🛡️ Cơ Chế Bảo Vệ Hiện Đại

🕯️
Stack Canary

Giá trị bí mật đặt giữa buffer và return address. Nếu bị thay đổi → detect overflow.

gcc -fstack-protector
🎲
ASLR (Address Space Layout Randomization)

Địa chỉ stack/heap/library thay đổi ngẫu nhiên mỗi lần chạy → kẻ tấn công không đoán được.

🚫
NX Bit / DEP

Stack không có quyền thực thi (No-Execute) — shellcode trên stack không chạy được.

📦
PIE (Position Independent Executable)

Địa chỉ của bản thân chương trình cũng được random.

✅ Coding Best Practices

❌ KHÔNG dùng
gets(buffer);          // Không kiểm tra độ dài
scanf("%s", buffer);   // Nguy hiểm tương tự
strcpy(dst, src);      // Không kiểm tra kích thước
strcat(dst, src);      // Nguy hiểm tương tự
✅ NÊN dùng
fgets(buf, sizeof(buf), stdin); // An toàn
scanf("%29s", buffer);          // Giới hạn độ dài
strncpy(dst, src, sizeof(dst)); // Kiểm tra kích thước
